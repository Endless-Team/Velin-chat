## Come Contribuire

### Reportare Bug

Se trovi un bug, apri una issue con:

- Descrizione chiara del problema
- Steps per riprodurlo
- Comportamento atteso vs. effettivo
- Screenshots o logs (se disponibili)
- Informazioni sul sistema (OS, versione browser, ecc.)

### Proporre Nuove Funzionalit√†

Apri una issue di tipo "Feature Request" descrivendo:

- Il problema che la feature risolve
- La soluzione proposta
- Alternative considerate
- Mockup o esempi (se applicabili)

### Pull Requests

1. Fork il repository
2. Crea un branch per la tua feature (`git checkout -b feature/nome-feature`)
3. Committa le modifiche (`git commit -am 'Add new feature'`)
4. Push al branch (`git push origin feature/nome-feature`)
5. Apri una Pull Request

### Linee Guida Codice

- Usa TypeScript per type safety
- Segui le convenzioni ESLint del progetto
- Scrivi codice pulito e auto-documentante
- Aggiungi commenti per logica complessa
- Testa le tue modifiche prima di pushare

### Commit Messages

Usa commit messages descrittivi:

````

feat: aggiungi supporto per videochiamate
fix: risolvi bug sincronizzazione messaggi
docs: aggiorna README con nuove istruzioni
style: formatta codice con Prettier
refactor: ottimizza componente Chat
test: aggiungi test per autenticazione

```
